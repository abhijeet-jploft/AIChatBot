Third-party website simulation

JP Loft chat integrated exactly like an external client site.

This page demonstrates both integration paths your clients can use: script embed and slug-host iframe embed. It ships with JP Loft credentials as defaults, so this works out of the box for live verification.

Mode 1: Script embed

Loads chat-widget.js directly into a third-party page with company context and API key headers.

Mode 2: Iframe embed page

Loads the server route /embed/{slug}/{embed secret}?companyId=… — secret is in the path; company id must match.

How To Integrate Across Your Whole Website

        

          Follow these steps once in your shared template/layout so chat is available on every page.
        


        
    
          
  1. Open Admin → Settings → Website embed and copy slug, company id, and embed secret (used as API key in script mode and in the iframe URL path).
    2. 
          
  2. Set your app origin correctly in this lab (production origin in production, dev origin in local testing).
    3. 
          
  3. Choose one install path: Script mode (recommended for launcher widget) or Iframe host mode (strict credential URL).
    4. 
          
  4. Paste the chosen snippet once into your global website template, just before </body>.
    5. 
          
  5. For SPA sites (React/Vue/Next): place it in the root layout so it does not reload on every route change.
    6. 
          
  6. For CMS sites (WordPress/Shopify/Webflow): paste into site-wide footer or global custom-code settings.
    7. 
          
  7. CORS: the API uses open CORS by default so third-party origins can call /api. Lock this down in production if you allowlist domains.
    8. 
          
  8. QA before go-live: test homepage, inner pages, mobile, and one lead capture flow end-to-end.
    9.
