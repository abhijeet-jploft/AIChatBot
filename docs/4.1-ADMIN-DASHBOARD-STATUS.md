# 4.1 Admin Dashboard — Implementation Status (JP Loft spec)

**Status: Done.** The Dashboard home page matches the Feature listing (AI Chat Agent / JP Loft doc) and loads after login.

---

## Layout structure

| Section | Spec | Implementation |
|--------|------|----------------|
| **System Status Header** | Agent name, status, domain, last training, language, voice | ✅ Agent name + edit icon; status **Online / Paused / Training** (Training when scrape job in progress); domain, last training, languages, voice |
| **Key Performance Metrics (KPIs)** | 6 cards | ✅ Visitors Engaged, Conversations Started, Leads Generated, Meetings Requested, Conversion Rate, AI Response Rate |
| **Live Activity Panel** | Real-time activity | ✅ Active visitors, Currently chatting, Last message, Current pages (WebSocket + fallback poll) |
| **Lead Snapshot** | Recent leads, quick actions | ✅ Name, requirement, source page, time; **Add note** (modal), **Mark contacted**, Open chat (new tab, scroll to lead), View lead |
| **Conversation Snapshot** | Last conversations | ✅ First message, duration, lead captured, status; View lead / Open chat (new tab); View all → Conversations page |
| **AI Insights (Smart Summary)** | Auto-generated daily summary | ✅ `aiInsights` from dashboard API; e.g. recent interest, hot leads, leads today, fallback message |
| **Recent Notifications** | System alerts, clickable | ✅ New lead, reminder overdue, reminder due today, **Training completed** (after save); links to `/admin/leads` or `/admin/training` |

---

## Actions (per spec)

- **Pause AI / Resume AI** — ✅ Buttons in header, PATCH `/admin/agent/status`.
- **Refresh** — ✅ Refresh button; dashboard data refetched.
- **Refresh training** — ✅ Training link with title "Refresh training / Open knowledge base".
- **Training** — ✅ Link to Training; status shows "Training" when scrape job is in progress.
- **View all (Leads)** — ✅ Link to `/admin/leads`.
- **View all (Conversations)** — ✅ Link to `/admin/conversations`.
- **Open chat** — ✅ New tab with session + companyId; optional scroll-to-lead.
- **View lead** — ✅ Link to lead detail.
- **Notifications** — ✅ Clickable links to relevant module.

---

## Refresh & update

- **Auto refresh** — ✅ Every 30s (configurable).
- **Manual refresh** — ✅ Refresh button.
- **Real-time** — ✅ WebSocket for live visitors; fallback poll every 15s.

**Live alerts (toast)**

- **New lead captured** — ✅ Pushed over WebSocket when a lead is captured; toast shown on dashboard.
- **Meeting requested** — ✅ Same push, message "Meeting requested — new lead captured" when intent is meeting_booking.
- **New chat started** — ✅ Toast when "currently chatting" count increases (from live data).
- **New visitor on site** — ✅ Toast when active visitor count increases (from live data).

---

## Empty state

- **No visitors yet** — ✅ Message: “Your AI agent is live and ready. Share your website to start receiving visitors.” + links to Training and Test chat.
